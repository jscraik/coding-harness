# Support

## Getting help

Before opening a new work item, check the existing resources:

- **[README](./README.md)** — setup, workflows, and command reference.
- **[CONTRIBUTING.md](./CONTRIBUTING.md)** — branching, gate requirements, and review workflow.
- **[docs/agents/](./docs/agents/)** — operational SOPs for agent-driven development.

## Reporting bugs and requesting features

This repository uses **Linear-first** intake. Create or update work items in the
[coding-harness project](https://linear.app/jscraik/project/coding-harness-bb735dbbda79).

Do **not** open GitHub Issues for bugs or feature requests — they will be
redirected to Linear.

## Security vulnerabilities

**Do not report security issues through public channels.**

Email **jscraik@brainwav.io** directly. See [SECURITY.md](./SECURITY.md) for
the full disclosure policy and response timeline.

## Direct contact

For questions that do not fit a Linear issue, email **jscraik@brainwav.io**.
