# PU-011 Implementation Intent

## Intent

Turn `root_surface_tidy` from a prose closeout claim into a receipt-backed
production claim supported by a narrow root-hygiene classifier. This slice
should classify the current repository root and prove delivery-truth only
accepts current claim-support evidence from that classifier.

This is not a destructive cleanup slice. It must not delete, move, or rewrite
root artifacts. The implementation should make future cleanup mechanically
verifiable by producing `root-hygiene-classification/v1` evidence that
delivery-truth can trust.

## Implementation Rules

1. Keep the root-hygiene classifier as a deep module with a small public facade,
   typed contract, validation helpers, and focused tests.
2. The classifier must separate at least these classes: `canonical_root`,
   `should_move`, `generated_tracked_intentionally`, and `legacy_drift`.
3. Passing `root_surface_tidy` evidence requires a current
	`root-hygiene-classification/v1` receipt produced by the classifier, marked
	`claim_support`, carrying a module-private verifier-owned runtime report
	token on a frozen report graph and a matching non-path repository identity,
	with no unclassified entries and no `legacy_drift` entries.
4. `should_move` entries may remain non-blocking only when the classification
   contract explicitly defers them and the classifier records that deferral.
5. Stale, missing, orientation-only, loose document-only, untrusted producer,
	unclassified-root, legacy/drift, post-token-mutated, direct-import-forged,
	digest-consistent copied evidence not produced by the verifier seam, missing
	repository identity, or cross-repository replay evidence must block
	`root_surface_tidy`.
6. `ARCHITECTURE.md` must map `src/lib/root-hygiene/**` if the module is added.

## Acceptance Criteria

- `root-hygiene-classification/v1` can classify the current tracked root into
  canonical, should-move, generated-intentionally-tracked, legacy/drift, and
  unclassified states.
- The classifier emits an evidence receipt suitable for delivery-truth only when
  classification is current, produced by the trusted classifier, and claim
  support is allowed.
- `root_surface_tidy` blocks on missing classification, stale classification,
  orientation-only evidence, loose document-only references, untrusted producer,
  unclassified entries, legacy/drift entries, post-token mutation attempts, and
  stale-head receipts when the verdict is head-bound.
- `root_surface_tidy` blocks when repository identity is missing or when
  root-hygiene evidence is replayed from a different repository context.
- `should_move` entries are reported but do not block when explicitly deferred by
  `docs/architecture/root-surface-classification.md`.
- `ARCHITECTURE.md` maps the new root-hygiene deep module if it is created.
