# Architecture

SynAIpse works around Codex rather than rebuilding it. Codex owns execution,
tools, repository operations, sessions, permissions, skills, and native review.
SynAIpse owns Jamie-specific lifecycle policy, evidence admission, project
adoption, and improvement decisions.

Target repositories retain authority over their code, architecture,
instructions, developer documentation, validation, privacy, and rollback.
Projects refer to external operator context using logical project and context
identifiers, never workstation paths or copied private documents.
