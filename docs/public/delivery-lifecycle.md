# Delivery Lifecycle

SynAIpse guides work through seven stages:

1. Shape the outcome, non-goals, and decision boundary.
2. Admit the work, owner, evidence, privacy, and rollback contract.
3. Build the smallest accepted change.
4. Prove the changed behavior and required validation lanes.
5. Review with independent evidence and external reviewer disposition.
6. Integrate only under standing authority and current evidence.
7. Improve by retaining, changing, consolidating, or deleting what the run
   taught the system.

Local tests, hosted CI, review state, tracker state, publication, merge state,
and deployment remain separate evidence lanes until a contract joins them.
