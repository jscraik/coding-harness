## Purpose

.harness/evals stores curated evaluation plans, fixtures, and result summaries.

## Authority

Eval artifacts are secondary-context unless a plan, spec, quality criterion, or
gate names them as required validation evidence.

## Use

Keep fixtures small, replayable, and tied to acceptance criteria. Store bulky or
volatile run output outside tracked repo truth unless explicitly promoted.
