Effect boundary: Effect is allowed here, not everywhere.
Agent-safe work area: a focused internal box with a guarded public surface.
Executable guard: tests that fail if the boundary drifts.

1. The Architecture In One Sentence

Callers stay on small facades; agents work inside deeper modules; tests guard the boundary. PR closeout is the main deep module. Doctor, command catalog, verify-work, memory-gate, HE phase-exit trust, output normalisation, and review-gate control surfaces are split into smaller work areas. Effect is contained to approved files. Linear workflow, Linear gate, PR template gate, rule lifecycle gate, policy gate, replay CLI args, plan-gate, gardener, memory-gate, silent-error, brainstorm-gate, drift-gate, evidence verify, review gate, and CodeRabbit review evidence mapping now live behind focused command adapters.

10. Why These Splits Exist

Deep module boundary

A small public interface hides richer internals. That lowers the context an agent needs and gives the human a clean review point.

Effect boundary

Effect is useful for typed failure and sequencing, but only inside approved modules so it does not leak through the whole app.

Executable guard

The boundary is real only when tests fail on drift. The ratchets and import guards stop the files quietly growing back together.
