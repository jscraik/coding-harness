Module Layout

Read this as a control map, not a wiring diagram. Blue boxes are the public places callers use. Green and teal boxes are safe work areas for agents. Purple is where Effect is allowed. Amber is what keeps the boundaries honest.

How To Read It

Public facade: the small interface callers should use.
Internal module: implementation hidden behind the facade.
Effect boundary: Effect is allowed here, not everywhere.
Doctor seam: a focused doctor command work area.
Executable guard: tests that fail if the boundary drifts.

1. The Architecture In One Sentence

Callers stay on small facades; agents work inside deeper modules; tests guard the boundary. PR closeout is the main deep module. Doctor is being split into smaller command seams. Effect is contained to approved files.

9. Why These Splits Exist

Deep module boundary

A small public interface hides richer internals. That lowers the context an agent needs and gives the human a clean review point.

Effect boundary

Effect is useful for typed failure and sequencing, but only inside approved modules so it does not leak through the whole app.

Executable guard

The boundary is real only when tests fail on drift. The ratchets and import guards stop the files quietly growing back together.
