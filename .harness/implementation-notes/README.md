## Table of Contents

## Purpose

.harness/implementation-notes captures dated execution notes, admissions, and
implementation evidence.

## Authority

Implementation notes are secondary-context unless an active plan, spec, decision,
or validator promotes a rule from them.

## Use

Use these notes to preserve why work changed shape, what was validated, and where
follow-up ownership lives when a blocker is outside current authority.
